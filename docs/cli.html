CLI Reference

The Weblisk CLI is a single standalone binary — no runtime, no package manager, no dependencies.

Commands

weblisk new

Create a new project:

bash 
weblisk new my-app
weblisk new my-app --local
weblisk new my-blog --template blog
weblisk new my-dashboard --template dashboard
weblisk new my-docs --template docs

| Flag | Description |

|------|-------------|

| --template | Use a starter template (blog, dashboard, docs) |

| --local | Include framework files locally instead of CDN |

weblisk build [root]

Build a production-ready static site:

bash 
weblisk build
weblisk build --minify
weblisk build --fingerprint
weblisk build --minify --fingerprint

The build copies your project to dist/ as-is. Opt-in optimizations:

| Flag | What it does |

|------|-------------|

| --minify | Minifies HTML, CSS, and JavaScript |

| --fingerprint | Hashes asset filenames for cache busting |

No composition, no transformation — what you write is what ships. The build only copies files and optionally optimizes them.

Output directory: dist/ (configure via WL_DIST in .env)

weblisk add page

Add a new page to an existing project:

bash 
weblisk add page contact
weblisk add page blog/post

Creates a full standalone HTML page at app/.html with import map and shell script included.

weblisk add island

Add a new island script:

bash 
weblisk add island carousel

Creates app/js/islands/.js with an export default function skeleton.

weblisk help

Show available commands and usage.

Configuration

.env

bash 
WL_ORIGIN=http://localhost:5500   # Base URL
WL_PORT=5500                       # Dev server port
WL_DIST=dist                       # Build output directory
