CLI Reference

The Weblisk CLI is a single standalone binary — no runtime, no package manager, no dependencies. Two command surfaces: Development (scaffold, generate, run) and Operations (manage a running hub).

Command Groups

GroupSurfacePurpose
ProjectDevelopmentScaffold, dev, build, vendor, doctor, secrets
ServerDevelopmentGenerate and run hub components
BlueprintDevelopmentUpdate, validate, apply patterns
IdentityDevelopmentML-DSA-65 operator keys and registration
StatusOperationsHub health overview
AgentsOperationsList, describe, deregister agents
DomainsOperationsList and describe domain controllers
WorkflowsOperationsList and describe active workflows
ApprovalsOperationsHuman-in-the-loop accept/reject
StrategiesOperationsGoal tracking and lifecycle loop
ObservationsOperationsLifecycle loop metrics and trends
FederationOperationsPeer trust, data contracts
OperatorsOperationsManage operator access and roles
AuditOperationsStream/query audit log
MarketplaceOperationsSearch, buy, publish capabilities
TestDevelopmentConformance tests and mock orchestrator
DeployOperationsRollback deployments
DependenciesDevelopmentAudit deps for vulnerabilities
PolicyDevelopmentValidate and test gateway policies

All operations commands require an authenticated operator identity (weblisk operator init + operator register). All commands support --json for machine-readable output.

Project Commands

weblisk new <name>

Scaffold a new project from weblisk-templates:

Terminal 
$ weblisk new my-hub --template server/starter
✓ Created my-hub/ from weblisk-templates/server/starter
✓ .gitignore generated (secrets, keys, tokens excluded)
✓ Run: cd my-hub && weblisk doctor

# Combine multiple templates (files are merged)
$ weblisk new my-app --template client/blog --template server/starter
FlagDescription
--template <path>Template path (e.g. server/starter, client/blog). Repeatable.
--localInclude client framework files locally instead of CDN
--lib <path>Custom path for local framework files

Template resolution: local project → configured template sources → weblisk-templates core.

weblisk dev

Start a development server with file watching and live reload:

Terminal 
$ weblisk dev --port 3000
✓ Dev server running at http://localhost:3000
  Watching for changes...
FlagDescription
--port <n>Port to serve on (default: 5500)

For client projects: file watching + live reload. For server projects: builds and runs the hub with restart-on-change.

weblisk build

Build production-ready output:

Terminal 
$ weblisk build --minify --fingerprint
✓ Built to dist/ (minified, fingerprinted)
FlagDescription
--minifyMinify HTML, CSS, and JavaScript
--fingerprintContent-hash asset filenames for cache busting

weblisk vendor

Download all client framework modules from CDN into your project for local use:

Terminal 
$ weblisk vendor --dest lib/weblisk
✓ Downloaded 52 modules to lib/weblisk/

weblisk doctor

Validate project health — checks config, .gitignore, secrets, and blueprint schemas:

Terminal 
$ weblisk doctor
✓ .gitignore excludes .weblisk/secrets/, .weblisk/keys/, .weblisk/token, .env
✓ .weblisk/config.yaml valid
✓ Secret file permissions correct (0600)
⚠ Secret 'LLM_API_KEY' declared by seo-analyzer but not set

weblisk secret

Manage agent secrets securely. Secrets are stored in .weblisk/secrets/{agent}/{KEY} with 0600 file permissions.

Terminal 
$ weblisk secret list
AGENT           SECRET        STATUS
seo-analyzer    LLM_API_KEY   ✓ set
cron            WEBHOOK_URL   ✗ missing

$ weblisk secret set cron WEBHOOK_URL
Enter value: ********
✓ Secret stored: .weblisk/secrets/cron/WEBHOOK_URL

$ weblisk secret get seo-analyzer LLM_API_KEY --confirm
⚠ WARNING: This prints a secret to the terminal.
sk-abc...xyz

$ weblisk secret rotate seo-analyzer LLM_API_KEY
Enter new value: ********
✓ Secret rotated: .weblisk/secrets/seo-analyzer/LLM_API_KEY
SubcommandDescription
listShow all declared secrets and their set/missing status
set <agent> <KEY>Set a secret (interactive prompt — never passed as argument)
get <agent> <KEY> --confirmPrint a secret (requires --confirm flag)
delete <agent> <KEY>Remove a secret file
rotate <agent> <KEY>Trigger rotation handler or prompt for new value

weblisk version

Print the CLI version and build info.

Server Commands

Generate and run hub components from YAML specs.

weblisk server init

Read all YAML specs and generate the complete hub implementation via LLM dispatch:

Terminal 
$ weblisk server init --platform go
Reading .weblisk/config.yaml, domains/*, agents/*...
Dispatching to LLM for code generation...
✓ Generated orchestrator, gateway, 1 domain, 2 agents
✓ Platform: Go
FlagDescription
--platform <p>Target platform: go (default), cloudflare, node, rust

weblisk server start

Build and start all hub components declared in .weblisk/config.yaml:

Terminal 
$ weblisk server start
✓ Orchestrator running on :9800
✓ Gateway running on :9900
✓ Domain 'seo' running on :9700
✓ Agent 'seo-analyzer' registered

weblisk server verify

Confirm orchestrator health and all registered components.

weblisk agent create <name>

Generate agent code from agents/{name}/agent.yaml spec via LLM dispatch:

Terminal 
$ weblisk agent create seo-analyzer --platform go
✓ Generated agent code: agents/seo-analyzer/main.go

weblisk agent start <name>

Build and run a single agent.

FlagDescription
--orch <url>Orchestrator URL
--port <n>Port for agent to listen on

weblisk agent verify <name> / weblisk agent list

Verify an agent is healthy or list all locally scaffolded agents.

weblisk domain create <name> / weblisk domain start <name>

Generate domain controller code from domains/{name}/domain.yaml and run it.

weblisk gateway create / weblisk gateway start

Generate and run the application gateway from hub configuration.

Identity Commands

Operator identity uses ML-DSA-65 (post-quantum). No passwords, no shared secrets.

weblisk operator init

Generate an ML-DSA-65 key pair for operator authentication:

Terminal 
$ weblisk operator init
Enter passphrase (min 12 characters): ********
Confirm passphrase: ********
✓ Key pair generated:
  Private: ~/.weblisk/keys/operator.key (encrypted, 0600)
  Public:  ~/.weblisk/keys/operator.pub

Private key is encrypted with Argon2id + AES-256-GCM. Passphrase cannot be skipped or passed as a CLI argument. Use --force to overwrite existing keys.

weblisk operator register

Register your public key with an orchestrator to obtain an access token:

Terminal 
$ weblisk operator register --orch http://localhost:9800
✓ Registered as admin (first operator — auto-approved)
  Token stored: ~/.weblisk/token (expires in 24h, auto-refresh)

weblisk operator token

Display or refresh the current token (auto-refreshes when < 1 hour remaining).

weblisk operator rotate

Rotate the operator key pair (registers new public key, invalidates old).

Agent Operations

weblisk agents list

List all registered agents on the hub.

weblisk agents describe <name>

Full detail for a single agent — capabilities, metrics, health, domain assignment.

weblisk agents deregister <name>

Remove an agent from the orchestrator. Requires --confirm or interactive name confirmation. Running tasks are drained before removal.

Domain Operations

weblisk domains list / weblisk domains describe <name>

List or inspect domain controllers — shows assigned agents, active workflows, and routing rules.

weblisk strategies list

ID         NAME                       STATUS    PRIORITY  PROGRESS
strat-001  Improve organic traffic    active    1         67%
strat-002  Reduce page load time      active    2         45%

weblisk strategies create / describe <id> / update <id> / delete <id>

Full CRUD for strategies. delete archives (admin only, requires confirmation). Archived strategies stop generating workflows but remain queryable.

Operator Commands

Manage operator access (admin only).

weblisk operators list

Terminal 
$ weblisk operators list
NAME    ROLE     STATUS    REGISTERED
alice   admin    active    2026-01-15T09:00:00Z
bob     viewer   revoked   2026-02-20T14:30:00Z
carol   admin    active    2026-03-01T11:00:00Z

weblisk operators describe <name>

Full operator detail — role, status, key ID, last active time.

weblisk operators revoke <name>

Revoke an operator's access. Invalidates their public key and all tokens. Cannot revoke yourself or the last admin.

weblisk operators role <name> <role>

Change an operator's role. Valid roles: admin, operator, auditor, viewer. Cannot demote yourself.

weblisk marketplace describe <id>

Full listing detail — capabilities, data contract, jurisdiction, pricing, reviews.

weblisk marketplace install <id>

Download and generate a purchased installable asset (not for live capabilities).

weblisk marketplace list

List active purchases and subscriptions with 30-day costs.

Publish a capability or asset to the marketplace from a listing config YAML.

Update pricing/metadata or delist a published listing. Delisting notifies active buyers; existing contracts honored until expiry.

weblisk marketplace collaborations / usage <id> / terminate <id>

Manage active collaborations (live capability connections). terminate initiates a 7-day grace period.

Policy Commands

weblisk policy validate

Validate the gateway policy file (policies.yaml) for syntax and semantic errors — checks for conflicts, unreachable rules, and missing role references.

Run policy assertions from policies_test.yaml against sample requests. Exit code 1 if any assertion fails.

Configuration

Config resolution order (highest priority first):

  1. Command-line flags: --orch http://localhost:9800
  2. Environment variable (implementation-defined)
  3. Project config: .weblisk/config.json
  4. User config: ~/.weblisk/config.json
.weblisk/config.json 
{
  "orchestrator_url": "http://localhost:9800",
  "operator_name": "alice",
  "default_format": "table"
}

Authentication Flow

Every operations command that calls the orchestrator follows this flow:

  1. Load operator private key from ~/.weblisk/keys/operator.key
  2. Load token from ~/.weblisk/token
  3. Check expiry — auto-refresh if < 1 hour remaining
  4. Set Authorization: Bearer <token> on request
  5. On 401 → refresh token → retry once
  6. If refresh fails → print "Session expired. Run: weblisk operator register"

Exit Codes

CodeMeaning
0Success
1General error (invalid arguments, missing config)
2Authentication error (no keys, expired token, rejected)
3Connection error (orchestrator unreachable)
4Not found (agent, workflow, strategy not found)
5Permission denied (role insufficient)
