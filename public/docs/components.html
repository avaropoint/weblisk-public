Components

Weblisk components are standard Custom Elements with a thin signal bridge. No Shadow DOM. No template compiler. No build step.

Counter Component

A reactive counter component with increment, decrement, and reset props — built on signal-backed attributes.

Counter

Click +, −, or Reset to see reactive state updates.

0 
Click +, −, or Reset
import { component } from 'weblisk';

component('wl-counter', {
  props: { start: 0 },
  setup({ props, $, effect }) {
    effect(() => {
      $('.count').textContent = props.start();
    });
    $('.inc').onclick = () => props.start.set(props.start() + 1);
    $('.dec').onclick = () => props.start.set(props.start() - 1);
  }
});
<wl-counter start="5">
  <button class="dec">-</button>
  <span class="count">5</span>
  <button class="inc">+</button>
</wl-counter>

Live Clock

A clock component with setup() and cleanup() lifecycle hooks — interval is cleared on disconnect.

Live Clock

Start the clock to see lifecycle in action.

--:--:--
Start the clock to see lifecycle in action
component('wl-clock', {
  setup({ $, cleanup }) {
    const tick = setInterval(() => {
      $('.time').textContent =
        new Date().toLocaleTimeString();
    }, 1000);
    cleanup(() => clearInterval(tick));
  }
});

component('wl-toggle', {
  props: { active: false },
  setup({ props, $, effect, emit }) {
    $('button').onclick = () => {
      props.active.set(v => !v);
      emit('change', { active: props.active() });
    };
  }
});

Component API

opts.propsObjectDefault values — type determines attribute parsing (string, number, boolean)
opts.setup(ctx)functionCalled on connectedCallback — receives context object
opts.template()functionOptional fallback DOM if the element is empty

Setup Context

PropertyTypeDescription

Events

Use emit to dispatch standard CustomEvents that bubble:

js 
component('wl-color-picker', {
  props: { value: '#000000' },
  setup({ props, $, emit }) {
    $('input').oninput = (e) => {
      props.value.set(e.target.value);
      emit('color-change', { color: e.target.value });
    };
  }
});

Template Fallback

If the element is empty, an optional template function provides initial DOM:

js 
component('wl-spinner', {
  template() {
    return '<div class="spinner" role="status" aria-label="Loading"></div>';
  },
  setup({ el }) {
    // Spinner is already visible — no further setup needed
  }
});

If the element already has children (server-rendered), the template is skipped.
