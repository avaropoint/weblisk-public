What is Weblisk?

Weblisk is a specification-driven framework. You define your requirements as blueprints — structured specifications that capture business logic, security models, data contracts, and deployment targets. Domain-expert agents consume those blueprints and generate production code for your chosen language and platform — code that you fully own.

The framework includes a governance layer with non-bypassable enforcement boundaries — four checkpoints (message, storage, external, response) inspect every operation regardless of agent cooperation. Nothing enters or exits your system without explicit consent.

Framework Overview

Weblisk has two main components:

1. The Specification Framework

97+ specifications across 7 categories define every aspect of how software is built:

  • Blueprints — Structured specifications that define your system. Anyone can read them — founders, product managers, developers.
  • Agents — 11+ infrastructure agents plus unlimited domain agents. Sandboxed, portable, AI-ready.
  • Governance — Non-bypassable enforcement boundaries, monotonic security, behavioral fingerprinting.
  • Federation — ML-DSA-65 post-quantum signing, hub-to-hub cryptographic trust, data contracts.
  • Migrations — Blueprint-versioned, agent-orchestrated upgrades. Every change diffed, validated, reversible.

2. The Client Framework

A zero-dependency JavaScript framework built on native browser standards:

  • Core — Signals, Islands, Hydration, Scheduler, Workers, DevTools
  • UI — 13 component utilities (dialog, toast, tabs, accordion, tooltip, dropdown, carousel, progress, skeleton, virtualList, clipboard, theme, component)
  • Data — Local-first IDB store with background sync
  • State — Forms, undo/redo, cross-tab sync, IDB-backed signals
  • Navigation — Route guards, scroll restoration, view transitions
  • Networking — Enhanced fetch, SSE, WebSocket, WebTransport
  • Security — CSP, CSRF, sanitization, Trusted Types, permissions
  • Performance — Web Vitals, timing marks, beacon reporting
  • PWA — Offline state, push notifications, manifest
  • Accessibility — ARIA, focus management, reduced motion
  • Testing — DOM assertions, signal spies, island test harness

Quick Example

Terminal

✓ Weblisk CLI installed

$ weblisk init my-app --template server
✓ Created my-app/ from weblisk-templates/server

$ weblisk generate --lang go
✓ Generated 24 endpoints, 3 agents, 7 patterns

$ weblisk dev
✓ Dev server running at http://localhost:3000
