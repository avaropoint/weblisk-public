PWA

Offline support, push notifications, background sync, and installability. Progressive web app primitives that work with any service worker.

Web App Manifest

Generate and inject a manifest at runtime — no static JSON file needed. Supports dynamic theme color updates.

Runtime Manifest

Web App Manifest generated and injected at runtime — no static JSON file needed.

Manifest injected at boot (see <head>)
import { manifest, injectManifest, setThemeColor } from 'weblisk/pwa/manifest.js';

const m = manifest({
  name: 'My App',
  short_name: 'App',
  start_url: '/',
  display: 'standalone',
  theme_color: '#2563eb',
});

injectManifest(m);
setThemeColor('#1e293b'); // e.g., dark mode

Manifest API

ExportTypeDescription
manifest(opts)functionCreate a manifest object from options
injectManifest(m)functionInject a <link rel="manifest"> with a blob URL
setThemeColor(color)functionUpdate the <meta name="theme-color"> dynamically

Connectivity & Sync

Reactive online/offline signal plus background sync engine. Mutations are queued locally and the service worker replays them when connectivity returns.

Connectivity & Sync

Reactive online/offline signal plus sync engine status. Toggle network in DevTools.

Checking...
import { online, syncStatus, pendingCount, triggerSync } from 'weblisk/pwa/offline.js';

effect(() => {
  if (!online()) showBanner('You are offline.');
});

// Manual sync
await triggerSync('todos');  // Sync a specific collection
await triggerSync();         // Sync all collections

Offline API

ExportTypeDescription
online()signalReactive boolean — true when connected
syncStatus()signal'idle', 'syncing', or 'error'
pendingCount()signalNumber of mutations queued for sync
triggerSync(tag?)asyncManually trigger background sync
registerPeriodicSync(tag, interval)asyncRegister periodic background sync (Chromium)

Push Notifications

Request permission and show local or push notifications. The permission export is a reactive signal that updates when the user responds to the prompt.

Push Notifications

Request permission and show local notifications.

Permission: checking...
import { requestPermission, subscribe, notify, permission } from 'weblisk/pwa/push.js';

// Reactive permission signal
effect(() => console.log('Permission:', permission()));

await requestPermission();
await notify('New message', {
  body: 'You have a new message from Alice',
  icon: '/icon-192.png',
});

Push API

ExportTypeDescription
permission()signalReactive notification permission state
requestPermission()asyncPrompt the user for notification permission
subscribe(vapidKey)asyncSubscribe to push via the service worker
notify(title, opts?)asyncShow a local notification
