Routing & Guards

Weblisk uses the browser's native navigation (MPA links) rather than client-side routing. Add guards and hooks to control navigation behavior, protect routes, and handle unsaved changes.

Route Guards

Protect routes with before/after navigation hooks. Guards can allow, block, or redirect navigation.

Route Guard

Enable "unsaved changes" then try to navigate — the guard blocks you.

No changes 
Toggle the checkbox, then click navigate
import { beforeNavigate, installGuards } from 'weblisk/nav/guard.js';

installGuards();

beforeNavigate((to, from) => {
  if (hasUnsavedChanges()) {
    return confirm('You have unsaved changes. Leave?');
  }
  return true;
});

Guard API

ExportTypeDescription
installGuards()functionInstall the navigation interceptor
beforeNavigate(fn)functionRegister a guard. Return true to allow, false to block, or a string to redirect
afterNavigate(fn)functionRegister a callback that fires after navigation completes

  sections.forEach(s => {
    s.hidden = s.dataset.section !== hash;

Scroll Restoration

Save and restore scroll positions across navigations:

js 
import { installScroll } from 'weblisk/nav/scroll.js';

installScroll({
  offset: 80,         // Account for sticky header
  savePositions: true, // Remember scroll position per page
});

Navigation Interceptor

Upgrade standard MPA link clicks into SPA-style content swaps without changing your HTML:

js 
import { intercept } from 'weblisk/nav/router.js';

const router = intercept({
  container: 'main',       // Swap this element's content
  transition: true,        // Use view transitions
  scroll: true,            // Scroll to top after swap
  exclude: ['/api/'],     // Skip these URL patterns
});

Uses the Navigation API where available, with click interception + popstate as a fallback.
