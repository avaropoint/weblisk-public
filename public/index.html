Blueprints

You already know what your business needs. A blueprint is how you write it down.

A blueprint is a structured description of what you want to build — your data models, security rules, integrations, and business logic — written in a format that both people and agents understand. No programming required. Define it once, generate from it in any language, for any platform. When requirements change, update the blueprint and regenerate.

Anyone Can Read It

Blueprints aren't code — they're structured descriptions anyone on your team can read. Founders, product managers, developers, and stakeholders all shape the same document. Your business logic is visible, not buried.

One Source of Truth

Security rules, data models, integrations, deployment targets — everything about your project lives in one versioned place. No scattered docs. No diagrams that drift from reality. One blueprint, always current.

Define Once, Generate Forever

No re-prompting. No lost context. The blueprint holds your requirements permanently — generate from it whenever you need, in any language, for any platform. Need to switch from Node to Go? Change the target, keep the blueprint.

Context That Doesn't Expire

AI chat memory fades between sessions. Blueprints don't. They hold your full system context — compliance checks, security boundaries, migration paths — so agents generate consistent, verifiable systems every time.

The Knowledge Stays

People move on — roles change, contributors rotate. The blueprint holds every decision and why it was made. Any person or AI can pick up where the last one left off. Your institutional knowledge lives in the spec, not in someone's head.

Upgrades Without Rewrites

When your system evolves, the blueprint evolves with it. Every migration is versioned, validated, and reversible. Update the spec, regenerate, deploy — no rewrite required.

The Workflow

Pick a domain. Describe it. Ship it.

Whether you're starting fresh or wrapping an existing system, the workflow is the same: describe what you need, let agents generate it, validate before it ships, and deploy alongside your existing tools — or replace them one at a time.

1

Initialize

One command scaffolds your project with blueprint structure, agent configs, and platform targets.

weblisk init my-project
2

Source Blueprints

Pull from the open marketplace, fork community blueprints, or write your own. Compose patterns, mix, extend anything.

weblisk add patterns/auth-token
3

Build & Generate

Generate production code for any supported language or platform. Built-in targets ship today — custom generators extend the rest. One blueprint, infinite targets.

weblisk generate --lang go
4

Transform & Migrate

Blueprint-versioned, agent-orchestrated migrations across your entire stack — data, infrastructure, agents, and configuration. Every change diffed, validated, and reversible.

weblisk migrate --validate
5

Preview & Validate

See exactly what ships before it goes live. Schema validation, security checks, quality gates.

weblisk preview --validate
6

Deploy & Monitor

Validated, versioned, deployed — with schema checks and automatic rollback. The loop never stops.

weblisk deploy --auto-rollback
Architecture

Seven layers. All open. All yours.

Weblisk exposes every layer — from wire protocol to marketplace — as open and specified. Each layer can wrap, connect to, or replace parts of your existing systems through governed data contracts. You decide which layers to use, extend, or swap out. The architecture is transparent because you should understand the systems you depend on.

Protocol

HTTP + JSON wire format that standardizes communication across every Weblisk project. Standard API contracts, consistent request/response shapes, and versioned endpoints — ensuring every agent, service, and hub speaks the same language.

Protocol specs →

Federation

Post-quantum identity signing (ML-DSA-65, FIPS 204) with hub-to-hub cryptographic trust verification. Federation protocol enables collaboration across independent hubs — every message signed, every agent verified, every data contract enforced. Quantum-resistant by default.

Federation specs →

Schemas & Validation

Five compliance tiers — from sandbox to certified. Every blueprint runs through a 4-phase validation pipeline with 50+ checks before it can register, publish, or interact with other agents. Content hashing detects post-generation tampering. Enterprise standards, enforced automatically — not by trust, but by architecture.

Agents

11 infrastructure agents plus unlimited domain agents. Each sandboxed, portable, and AI-ready. Agents register capabilities, execute tasks, and collaborate autonomously across your entire system.

Patterns

Governance

You decide what enters and exits your system.

AI is powerful — but power without boundaries is a liability. Weblisk's governance layer isn't a policy document. It's built into the architecture. Non-bypassable enforcement boundaries inspect every operation at every layer. Nothing enters or exits your system without explicit consent — not data, not agent actions, not external calls. You set the rules. The framework enforces them.

Non-Bypassable Enforcement

Four checkpoints — message, storage, external, response — inspect every operation regardless of agent cooperation. This isn't middleware that can be skipped. It's a boundary built into the architecture that agents cannot circumvent, period.

Explicit Consent Boundaries

Data doesn't move without permission. Agent actions don't execute without authorization. External services don't connect without policy. Every boundary crossing requires explicit consent defined in your blueprints — protecting your most valuable assets: your data and intellectual property.

Works With Any AI

Not locked to specific frameworks. Weblisk wraps LangChain, CrewAI, Google ADK, or any other AI tool — unmodified — while enforcing governance at every boundary. Use whatever AI you want. The governance layer ensures it operates within your rules.

Security That Only Gets Stronger

Monotonic security means policies can only add restrictions — never loosen them. No configuration change, no prompt, no agent can reduce your security posture. Once a boundary is set, it holds. This is how enterprise security should work.

Rogue Agent Detection

Behavioral fingerprinting identifies when agents deviate from expected patterns. Anomalous behavior triggers quarantine escalation — isolating compromised or malfunctioning agents before they can impact your system or your data.

Data Sovereignty

Your data stays where you put it. Blueprint-defined policies control exactly what enters and exits your organization. Cryptographic federation enables collaboration across trusted hub networks — with real data contracts, not handshake agreements.

Getting Started

Up and running in minutes.

Install the CLI, initialise a project, generate production code, and run it locally.

1

Install the CLI

One command. No dependencies. Works on macOS, Linux, and WSL.

Terminal 
$ curl -fsSL https://cdn.weblisk.dev/install.sh | sh
✓ Weblisk CLI installed to ~/.weblisk/bin/weblisk
2

Initialise your project

Start from scratch or source a starter template — client, server, or full-stack. Use either, or combine both.

Terminal 
$ weblisk init my-app --template server
✓ Created my-app/ from weblisk-templates/server
✓ Blueprint scaffolding ready
3

Define and compose Optional

Create domains, compose patterns, and shape blueprints to your exact requirements. Skip this if a starter already covers what you need.

Terminal 
$ weblisk domain create chat \
    --patterns api-ai,auth-token,realtime-chat
✓ Domain "chat" created with 3 patterns
4

Generate

Generate production code — client and server components built from your blueprints in your target language.

Terminal 
$ weblisk generate --lang go
✓ Generated 24 endpoints, 3 agents, 7 patterns
✓ Output: ./generated/
5

Run and deploy

Start a local dev server to watch your results, then deploy to your target platform when you're ready.

Terminal 
$ weblisk dev
✓ Dev server running at http://localhost:3000
✓ Watching for changes...

$ weblisk deploy --target cloudflare
✓ Deployed to production (validated, versioned)
