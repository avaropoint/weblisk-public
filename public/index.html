Blueprints

Software finally has what every building has always had.

An architect doesn't hand a construction crew a blank page — they hand them blueprints. Exact specifications of every wall, wire, and load-bearing beam. Software has never had that. Until now. Weblisk blueprints are schema-driven specifications that define exactly what your software does, how it's secured, where it deploys, and how every component connects — so agents can generate it to the exact standard you set.

Readable by Anyone

No programming knowledge required. Product managers, designers, founders — anyone can read and write a blueprint. It's structured specification, not source code.

Controls Everything

Design, security, integrations, data models, deployments, agent behavior — every aspect of your project, specified in one versioned corpus. Not fragments. 80+ specs that cross-reference and generate together.

Schema-Validated

Every blueprint is validated against Weblisk schemas with five compliance tiers — from sandbox to certified. Enterprise standards, enforced automatically. Nothing ships that doesn't meet the spec.

Eliminates AI Waste

Define once, generate forever. No re-prompting, no token churn, no hallucinated architecture. Blueprints give AI the structured context it needs — verifiable, reproducible, and cost-efficient.

Centralised Domain Knowledge

When a developer leaves, the knowledge stays. The specification is the handover — any team, any AI, any generation of developers can execute from it. No reverse-engineering. No tribal knowledge.

Safe, Versioned Upgrades

Specs evolve — and so do your data, schemas, and agents. Every deployment is diffed, validated, and reversible. Change management is built into the specification layer, not bolted on.

Generative UI/UX

From blueprint to production — every layer, meticulously generated.

Your blueprint defines the target. If it specifies a web page, agents generate semantic HTML, scoped CSS, and interactive JavaScript. If it specifies an API, they generate server code in Go, Rust, or Node.js. Mobile, edge, microservices — whatever the blueprint calls for, agents collaborate autonomously to generate every layer in a continuous feedback loop. The Workflow

From ideation to production — and every step in between.

This isn't a simplified demo. This is the real workflow — initialize, source, generate, migrate, validate, deploy — all orchestrated by agents that collaborate autonomously across every layer of your stack.

1

Initialize

One command scaffolds your project with blueprint structure, agent configs, and platform targets.

weblisk init my-project
2

Source Blueprints

Pull from the open marketplace, fork community blueprints, or write your own. Compose patterns, mix, extend anything.

weblisk add patterns/auth-token
3

Build & Generate

Generate production code in Go, Rust, or Node.js. Deploy to platforms like Cloudflare Workers. One blueprint, infinite targets.

weblisk generate --lang go
4

Transform & Migrate

Schema migrations, data transformations, and infrastructure transitions. Every change diffed, validated, and reversible.

schemas/*.yaml → migrations
5

Preview & Validate

See exactly what ships before it goes live. Schema validation, security checks, quality gates.

weblisk preview --validate
6

Deploy & Monitor

Validated, versioned, deployed — with schema checks and automatic rollback. The loop never stops.

weblisk deploy --auto-rollback

Getting Started

Up and running in three steps.

Install the CLI, define your first domain, and generate production code.

1

Install the CLI

One command. No dependencies. Works on macOS, Linux, and WSL.

Terminal 
$ curl -fsSL https://cdn.weblisk.dev/install.sh | sh
✓ Weblisk CLI installed to ~/.weblisk/bin/weblisk
2

Define your domain

Initialise a project and compose patterns into a domain blueprint.

Terminal 
$ weblisk init my-project
✓ Created my-project/ with blueprint scaffolding

$ weblisk domain create chat \
    --patterns api-ai,auth-token,realtime-chat
✓ Domain "chat" created with 3 patterns
3

Generate and deploy

Generate production code for your target language and platform.

Terminal 
$ weblisk generate --lang go
✓ Generated 24 endpoints, 3 agents, 7 patterns
✓ Output: ./generated/

$ weblisk deploy
✓ Deployed to production (schema validated, data preserved)
