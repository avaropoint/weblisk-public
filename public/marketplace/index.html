Marketplace

The decentralized marketplace for the hub network.

Every hub can be a seller, buyer, or both. Buy live capabilities over federation, download installable blueprints, publish agents, sell domain expertise — all governed by cryptographic identity and verified data contracts.

Listing Types

Five asset types. Two delivery models.

Live capabilities run on the seller's hub, invoked over federation. Installable assets download into your hub and run under your full control.

Capabilities

Live, invocable agent capabilities hosted on a provider hub. Your hub calls them over federation — no code to install, governed by data contracts.

Architecture

Two delivery models. One protocol.

The distinction matters: installable assets run inside your hub, under your full control. The seller has no ongoing access. Live capabilities run on the seller's infrastructure, governed by federation rules.

Delivery Models 
Live service (capability):
  Buyer hub ──federation──► Provider hub (runs the agent)
  • Governed by data contract
  • Pay per invocation or subscription
  • Zero code on buyer's infrastructure

Installable asset (blueprint, template, agent, data contract):
  Buyer hub ──download──► Blueprint files ──CLI generate──► Buyer's own code
  • Full ownership of generated code
  • One-time or subscription payment
  • Seller has no ongoing access
Commerce

Seven pricing models. From free to enterprise.

Sellers choose the model that fits their offering. Buyers review pricing and data contracts before purchase — all enforced by the protocol.

Model Description Best For
Free No charge — open source or community contribution Core blueprints, community agents
One-time Single payment for permanent access Blueprints, templates, standalone agents
Per-invocation Charge per federated task execution Live capabilities (high-volume, low-cost)
Monthly subscription Recurring fee for access Live capabilities (predictable workloads)
Tiered Price brackets based on volume Variable workloads
Revenue share Percentage of buyer's revenue from the capability Co-investment models
Custom Negotiated enterprise pricing Large partnerships
Workflow

Discover. Evaluate. Purchase. Deploy.

The entire purchase flow is protocol-native — signed by both parties, recorded on both hubs, and governed by the same enforcement layer as every other operation.

1. Discover

Search via the marketplace UI, CLI, or hub-to-hub federation. Filter by category, platform support, compliance tier, pricing model, or rating.

2. Evaluate

Review verified metrics (uptime, latency, rating), data contracts, pricing, platform support, and trial availability — all cryptographically attested.

3. Purchase

Buyer's hub signs the purchase request with its ML-DSA-65 key. For live capabilities, federation peering initiates automatically. For assets, download begins.

4. Deploy

Installable assets generate into your hub via CLI. Live capabilities become available in your workflows immediately. Dual-signed receipt recorded on both hubs.

CLI

Full marketplace access from the terminal.

Every marketplace operation is available via the CLI. Search, evaluate, purchase, install, publish, and manage listings without leaving your terminal.

Terminal 
# Search for capabilities
$ weblisk marketplace search "demand forecasting"
ID        NAME                        TYPE          SELLER        PRICE      RATING
mkt-001   Demand Forecasting Agent    capability    acme-ai       $0.02/req  4.8★
mkt-002   Supply Chain Optimizer      capability    logicorp      $0.05/req  4.5★
mkt-003   Inventory Predictor         installable   predict-co    $99/mo     4.2★

# Purchase a live capability
$ weblisk marketplace buy mkt-001 --accept-contract --accept-pricing
✓ Purchase confirmed.
  Type:   live capability
  Access: federation peering initiated with acme-ai
  Status: Active — capability available in workflows

# Install a purchased blueprint
$ weblisk marketplace install mkt-003
✓ Blueprint downloaded to agents/inventory-predictor/
  Run: weblisk agent create inventory-predictor --platform go
Sell

Publish your expertise. Earn revenue.

Turn your domain agents, blueprints, and capabilities into marketplace listings. Every listing is signed with your hub's ML-DSA-65 key and verified by the network.

Terminal 
# Publish a capability
$ weblisk marketplace publish --type capability --config listing.yaml

✓ Data contract parsed
✓ Pricing terms set

Verified Reviews

Only hubs that have purchased or invoked your listing can leave reviews. Every review is signed by the reviewer's ML-DSA-65 key — no fake reviews possible.

Seller Analytics

Revenue by listing and time period. Download counts, active installations, usage metrics for live capabilities, review trends, and payout history.

Update & Delist

Update pricing, metadata, or version at any time. Delist with a reason — active buyers are notified and existing contracts honored until expiry.

Registry

This site aggregates listings from registries that opt into public visibility. Every hub can also run its own private or partner marketplace.
