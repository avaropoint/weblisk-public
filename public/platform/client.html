Client Framework

A zero-dependency JavaScript framework built on native browser standards. Islands hydrate on demand, signals drive fine-grained reactivity, and the entire framework generates from blueprints.

Core

  • Signals & Reactivity — Fine-grained reactive primitives: signal, computed, effect, batch
  • Island Architecture — Independent interactive regions with isolated hydration
  • Progressive Hydration — Three strategies: load, visible, idle
  • Components — Declarative UI components with props, slots, and variants
  • Scheduler — Priority-based task scheduling with frame-budget awareness
  • Workers — Off-main-thread computation with structured messaging
  • DevTools — Signal inspector, island debugger, performance profiler

Data & State

  • State Management — Forms, undo/redo, cross-tab sync, IDB-backed signals
  • Local-First Store — IndexedDB-backed persistence with background sync to server agents

Navigation

  • Routing & Guards — Client-side routing with auth guards, redirects, and lazy loading
  • View Transitions — Native View Transitions API with fallback animations

Networking

  • Enhanced Fetch — Interceptors, retry, caching, request deduplication
  • Realtime — SSE, WebSocket, and WebTransport with automatic reconnection
  • Analytics — Privacy-first event tracking with beacon reporting

Security & Performance

  • Security — CSP, CSRF protection, input sanitization, Trusted Types
  • Performance — Web Vitals monitoring, timing marks, beacon reporting
  • PWA — Offline state, push notifications, manifest generation

UI & Accessibility

  • UI Components — Themeable component library with design token integration
  • Accessibility — ARIA patterns, focus management, reduced motion, screen reader support

Tools

  • CLI — Project scaffolding, blueprint validation, code generation
  • Testing — DOM assertions, signal spies, island test harness

For detailed implementation guides and API references, see the documentation.
