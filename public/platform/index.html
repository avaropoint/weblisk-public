Architecture

Seven layers. All open. All yours.

Weblisk exposes every layer — from wire protocol to marketplace — as open, specified, and interconnected. A non-bypassable enforcement boundary inspects every operation at four checkpoints. You decide which layers to use, extend, or replace.

Protocol

HTTP + JSON wire format that standardizes communication across every Weblisk project. Standard API contracts, consistent request/response shapes, and versioned endpoints — ensuring every agent, service, and hub speaks the same language.

Federation

Post-quantum identity signing (ML-DSA-65, FIPS 204) with hub-to-hub cryptographic trust verification. Federation protocol enables collaboration across independent hubs — every message signed, every agent verified, every data contract enforced. Quantum-resistant by default.

Schemas & Validation

Five compliance tiers — from sandbox to certified. Every blueprint runs through a 4-phase validation pipeline with 50+ checks before it can register, publish, or interact with other agents. Content hashing detects post-generation tampering.

Agents

11 infrastructure agents plus unlimited domain agents. Each sandboxed, portable, and AI-ready. Agents register capabilities, execute tasks, and collaborate autonomously across your entire system.

Patterns

40 cross-cutting contracts — API, auth, governance, webhooks, workflows, storage, deployment, and more. Each pattern generates real endpoints with real security and real validation. Policies can only add restrictions, never loosen them.

Platforms

Languages: Go, Rust, Node.js. Platforms: Cloudflare Workers today, with AWS, GCP, Azure adapters on the roadmap. One blueprint, any target.

Governance

AI without boundaries is a liability. We built the boundaries.

Every AI framework lets agents act. None of them enforce what agents can't do. Weblisk is the first framework where governance is architecture — non-bypassable enforcement boundaries inspect every operation at every layer.

Non-Bypassable Enforcement

Four checkpoints — message, storage, external, response — inspect every operation regardless of agent cooperation. Agents cannot circumvent these boundaries.

Explicit Consent Boundaries

Data doesn't move without permission. Agent actions don't execute without authorization. Every boundary crossing requires explicit consent defined in your blueprints.

Monotonic Security

Policies can only add restrictions — never loosen them. No configuration change, no prompt, no agent can reduce your security posture. Once a boundary is set, it holds.

Behavioral Fingerprinting

Identifies when agents deviate from expected patterns. Anomalous behavior triggers quarantine escalation — isolating compromised agents before impact.

Federation

Post-quantum cryptography. Hub-to-hub collaboration.

ML-DSA-65 (FIPS 204) post-quantum identity signing ensures your federation network is resistant to both classical and quantum attacks. Collaborate across independent hubs with cryptographic trust — every message signed, every agent verified, every data contract enforced.

ML-DSA-65 Signing

FIPS 204 post-quantum digital signatures. Quantum-resistant by default — every identity, every message, every agent action is cryptographically verified.

Hub-to-Hub Trust

Independent hubs collaborate through cryptographic trust verification. No central authority needed — each hub maintains sovereignty while participating in federated networks.

Data Contracts

Real data contracts — not handshake agreements. Blueprint-defined policies control exactly what enters and exits your organization across every federated connection.

Client Framework

Zero-dependency reactive islands. Built on browser standards.

The Weblisk client framework generates reactive UI from blueprints. Islands hydrate on demand — idle, visible, or immediate — with zero build dependencies. Every section on this site is a live island. Specification in, reactive UI out.

Island Architecture

Independent, self-contained components that hydrate on demand. No monolithic bundle. Each island loads only when needed — reducing initial payload to near-zero.

Signals & Reactivity

Fine-grained reactivity with signals. No virtual DOM. No diffing. Direct DOM updates through a dependency graph — faster and more predictable than framework abstractions.

Progressive Hydration

Three strategies — load, visible, idle — give you precise control over when JavaScript activates. Serve static HTML instantly, hydrate interactive behavior when it matters.

Security Built In

CSP-compliant by default. Strict output encoding, sandboxed execution, and no eval. Security isn't an add-on — it's how the framework generates code.
