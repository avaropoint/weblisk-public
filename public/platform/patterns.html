Patterns

37 cross-cutting contracts that agents extend and opt into. Unlike the enforcement layer (which is always active), patterns define reusable behaviors that generate real endpoints, real validation, and real security.

How Patterns Work

Patterns are YAML blueprints that agents reference via extends in their frontmatter. A pattern is a dependency that provides a known contract — its interface, behavior, configuration, and verification criteria — without prescribing a specific implementation language.

  • Agents declare which patterns they extend
  • Patterns generate real code: endpoints, middleware, validation
  • Patterns compose together with explicit dependency contracts
  • The enforcement layer ensures declared patterns are actually applied

Patterns define rules. The enforcement layer ensures they are applied. Agents that declare honest intents benefit from fast-path evaluation. Agents that lie, misbehave, or drift are detected and quarantined.
