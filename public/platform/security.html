Security & Governance

Every AI framework lets agents act. Weblisk is the first where governance is architecture — a non-bypassable enforcement layer inspects every operation at four structural boundaries. No agent escapes it.

The Enforcement Layer

The enforcement layer sits between agents and every resource they touch. It inspects operations at four structural boundaries:

  1. Message boundary — all inter-agent communication
  2. Storage boundary — all data persistence operations
  3. External boundary — all outbound network calls
  4. Response boundary — all data returned to clients

Design Invariants

  • Structural non-bypassability — agents MUST NOT have any path to resources that bypasses enforcement. This is a platform invariant, not a policy.
  • Verify both input and output — validates operations before execution AND responses after
  • Declared authority only — agents can only perform operations they declared in their manifest AND that match their active data contract
  • Fail-closed at every boundary — if enforcement cannot evaluate a request, the request is denied

The enforcement layer is not a policy engine (that's the policy pattern). It is not a safety classifier (that's the safety pattern). It is the structural mechanism that ensures those patterns are applied regardless of agent cooperation.

Six Trust Boundaries

The threat model identifies 6 trust boundaries — transitions where security context changes and enforcement must validate:

# Boundary Threat Surface
1 Browser ↔ App Gateway Untrusted external clients — XSS, CSRF, session hijacking, injection
2 App Gateway ↔ Agents Internal network — privilege escalation, unauthorized agent access, token forgery
3 Agents ↔ Storage Data persistence — SQL injection, scope violation, unauthorized table access
4 Operator ↔ Admin Gateway Privileged access — credential theft, config tampering, audit log manipulation
5 Hub ↔ Hub Federation Cross-organization — man-in-the-middle, trust exploitation, data exfiltration
6 Enforcement Layer Policy decision — bypass attempts, rule manipulation, timing attacks

Every boundary has defined threats, mitigations, and monitoring requirements specified in the threat model.

Monotonic Security

Security policies can only add restrictions, never loosen them. No configuration change, no operator command, no agent request can reduce the security posture of a running hub.

  • New restrictions: applied immediately
  • Removal of restrictions: requires hub restart with explicit operator authorization
  • Policy conflicts: the more restrictive policy always wins

Behavioral Fingerprinting

Agents are fingerprinted at registration. The enforcement layer maintains a behavioral profile for each agent based on:

  • Manifest declarations (capabilities, namespaces, collaborators)
  • Communication patterns (which agents it talks to, frequency, payload sizes)
  • Storage access patterns (which tables, read/write ratio, query patterns)
  • Event publishing patterns (namespaces, scopes, frequency)

Changes to manifest, capabilities, or behavioral patterns are detected and flagged before they can affect cross-boundary interactions. Anomalous behavior triggers investigation or quarantine.

Quarantine & Escalation

When an agent exhibits compromised or anomalous behavior, it can be quarantined before impact spreads:

  1. Detection — behavioral fingerprint drift, manifest violation, or enforcement boundary violation
  2. Isolation — agent loses messaging, storage, external access. Only health endpoint remains active.
  3. Investigation — audit log review, behavioral comparison, manifest inspection
  4. Resolution — agent is cleared and restored, or deregistered and flagged

Quarantine orders are published as system.agent.quarantined events. All other agents are notified immediately.

Post-Quantum Cryptography

Weblisk uses exclusively post-quantum algorithms. No RSA, no ECDSA, no Ed25519, no classical Diffie-Hellman.

Purpose Algorithm Standard
Digital signatures ML-DSA-65 (CRYSTALS-Dilithium) FIPS 204
Key encapsulation ML-KEM-768 (CRYSTALS-Kyber) FIPS 203
Backup signatures SLH-DSA-SHA2-128s (SPHINCS+) FIPS 205
Symmetric encryption AES-256-GCM FIPS 197
Key derivation Argon2id RFC 9106

Per NIST IR 8547, all elliptic curve and RSA algorithms will be deprecated by 2030 and removed by 2035. Weblisk is ready today.

Data Sovereignty

Federation data contracts ensure data never leaves its permitted jurisdiction:

  • Data minimization by construction — contracts declare the minimum viable data set. Fields not in the contract are stripped before crossing boundaries.
  • Sovereignty-aware routing — tasks involving regulated data are only routed to agents within the permitted jurisdiction.
  • Non-repudiation — every cross-boundary message is signed by both sending and receiving orchestrator. Full chain of custody is auditable.
  • Fail-closed boundaries — federation data contracts reject unknown fields, not ignore them.
