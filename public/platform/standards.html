Standards

Standards govern how developers build with the framework. They define the YAML vocabulary for describing projects — from project structure to page layouts to interactive islands. These are best practices, not hard requirements.

Standards vs Schemas

Schemas Standards
Nature Enforced specification Best practice guidance
Scope Framework internals Developer projects
Validation Machine-checked Developer judgment
Target Blueprint authors (framework) Project authors (users)

Standards Index

Guiding Principles

  1. Describe intent, not implementation. YAML says what you want. The LLM decides how to build it. Patterns constrain the how.
  2. Everything inherits from global. Brand colors, a11y level, SEO policy — declared once, applied everywhere.
  3. Escape hatches exist. Write code directly and mark managed: false. The framework respects manual overrides.
  4. Zero ambiguity. A new developer should read blueprints/ and understand the entire project.
  5. Composability over complexity. Many small blueprints > one giant blueprint.

The Generation Pipeline

There is no traditional build step. The LLM is the compiler. YAML is the source code. Production services — APIs, data models, business logic, and interfaces — are the compiled output.

  1. global.yaml — Project identity, brand, policies (loaded first, always)
  2. code.yaml — Code conventions (loaded second, constrains all output)
  3. theme.yaml — Design tokens derived from global brand
  4. Individual blueprints — Pages, components, islands, etc.

The LLM receives: global + code + theme + target blueprint + patterns and produces files that conform to all constraints simultaneously.

Three Project Modes

A developer picks their complexity. A portfolio is 3 blueprints. A SaaS app is 30. The structure scales because each blueprint is self-contained.

Mode What You Describe What You Get
Client-only global + code + pages Static HTML/CSS/JS, CDN import map
Client + Islands Above + islands Static site + interactive regions bound to agents
Full stack Above + domains + agents Complete hub with client and server
